COZ → Parquet → VSI → data
How It Works · Step 1

The problem with normal ZIP

A ZIP stores its file index, the Central Directory, at the end of the archive. For a dataset with one million samples and three files each, the CD alone weighs around 200 MB. To open anything remotely, you have to fetch those 200 MB from the tail first. This is why many people assume ZIP is slow for large datasets.

Cloud-optimized ZIP flips this

A small JSON index at byte 0 stores offsets to a handful of key files. In TACO, those are the Parquets in METADATA/ and COLLECTION.json. A few kilobytes instead of hundreds of megabytes.

Once you reach the Parquets, you have pointers to every data file in the archive. The CD at the end becomes irrelevant.

FOLDER mode skips this step.

How It Works · Step 2

Metadata loads once, access is instant

When you open a TACO dataset, the reader loads the Parquet files from METADATA/ into in-memory tables. Even for one million samples, the files are small on disk, so this takes a moment. It happens once.

After that, every sample is a direct row lookup. Filtering by cloud cover, split, or region scans an in-memory table. Random access in microseconds.

Each row represents a node in the sample tree. When the node is a file, the reader builds a VSI path, a pointer to the exact bytes on disk or cloud.

How It Works · Step 3

From filtered rows to raw bytes

Each row that passed the filter carries a VSI path. Think of it as a URL that works for anything, not just HTTP. /vsicurl/ reads over HTTP. /vsis3/ reads from S3. /vsisubfile/ reads a byte range from any source.

These paths are chainable. /vsisubfile/8234_1024,/vsicurl/https://hf.co/ds.zip tells the reader: connect to this URL, jump to byte 8234, read 1024 bytes. One path, only the bytes you need.

FOLDER mode is the same chain without /vsisubfile/. The files are individual, so there is no byte range to resolve. /vsicurl/https://hf.co/DATA/42/before/B02.tif works just as well.

How It Works · Step 4

From query to training loop

TACO slots naturally into ML pipelines. Query the metadata, filter the samples you need, iterate. Your dataloader receives VSI paths and reads them.

VSI paths work the same whether the dataset is on local disk or on a remote server. Download everything and train offline, or stream directly from cloud storage. The format works either way, you choose what fits your setup.
