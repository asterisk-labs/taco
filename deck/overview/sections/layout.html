Physical Layout

How it looks on disk

COLLECTION.json stores the contract and dataset-level fields like license, providers, and spatial extent. DATA/ holds the files — samples are numbered 0, 1, 2, …, N-1, and the path to any file is predictable from the index and the contract alone.

Metadata lives in Parquet files, one per level of the hierarchy. collection.parquet has one row per sample. sample.parquet has one row per child inside each sample. If the structure has subfolders like before/ and after/, each gets its own Parquet with one row per file inside it.

A TACO dataset can exist as a FOLDER (plain directory, supports incremental append) or as a cloud-optimized ZIP (single archive, immutable, byte-range accessible).
