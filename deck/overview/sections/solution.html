The Solution

Declare a contract, build your dataset

Before writing a single byte, you declare a contract that fixes the shape of your dataset. Every sample obeys this contract. It has two parts.

The structure (σ) defines the file tree inside each sample — which files and folders exist, their names, and their nesting. Every sample shares the exact same tree.

The metadata (μ) defines what fields exist at each level of the hierarchy. Each non-leaf level generates its own Parquet table, queryable with SQL — no server needed.

A TACO dataset is valid when every sample satisfies both: contract = σ ∧ μ.
