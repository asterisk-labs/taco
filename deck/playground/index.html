Planned construction

The writer computes every Local File Header and payload offset before the archive is written. That is why the byte-zero index can contain final offsets.

    CZIP index payload

    The first entry payload starts at byte 51 and stores names, offsets, and sizes in compact little-endian columns.

    ZIP compatibility

    The output remains a normal ZIP: every user file is a STORE entry, then the Central Directory and EOCD close the archive.

    Integrity patch

    The FNV-1a 64 value covers the index payload and final 32 KiB of the archive. It is a structural check, not a cryptographic signature.

    Client-side WASM path

    The C writer can run in the browser through Emscripten. This page automatically uses the compiled writer when the static WASM files are deployed, then falls back to the JavaScript explainer when they are not.

    1. Build cozip/javascript/wasm/cozip.js and cozip.wasm with Emscripten.
    2. Deploy them with cozip/javascript/src/cozip-wasm-wrapper.js on the same static site.
    3. Copy browser File bytes into WASM memory.
    4. Call cozip_wasm_write_archive_from_buffers, which runs plan, index serialization, archive write, padding, and hash patching in C.
    5. Read the output ZIP bytes from MEMFS and download them as a Blob.
